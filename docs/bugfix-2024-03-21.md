## Проблема #1: depth_limit не работает

### Описание
При генерации курса параметр `depth_limit` (принимает значения 2-5) должен влиять на глубину вложенности topics в структуре курса. Однако при любом значении depth_limit создаётся только один уровень topics с плоским списком уроков.

**Ожидаемое поведение:**
```
depth=3: Topic -> SubTopic -> Lessons
```

**Фактическое поведение:**
```
depth=3: Topic -> Lessons (все на одном уровне)
```

### Симптомы
- В БД курс с depth=3 не имеет ни одного SubTopic
- При depth=4, depth=5 результат идентичен depth=2

### Корневые причины (2 места)

#### Причина 1: course_outline.json prompt

**Файл:** `ai_backend_framework/ai-backend-framework/pipelines/course_outline.json`

**Текущий prompt:**
```
=== ТРЕБОВАНИЯ К СТРУКТУРЕ ===
Создай JSON объект с массивом topics. Каждый элемент должен содержать:
1. order_index — порядковый номер (0, 1, 2...)
2. title — название темы/урока
3. type — тип: "topic" для разделов, "theory" для теории, "practice" для практики
4. description — краткое описание
5. children — массив дочерних элементов с теми же полями
```

- Не сказано явно что sub-children ДОЛЖНЫ быть topics
- Проще создать плоскую структуру (все theory/practice на одном уровне)
- Пример в prompt одноуровневый

**Исправление:**

#### Причина 2: JsonTransformNode._transform_item()

**Файл:** `ai_backend_framework/ai-backend-framework/ai_flow_engine/nodes/json_parse_node.py`

**Текущий код (строки ~165-170):**
```python
def _transform_item(self, item: Dict[str, Any], default_type: str = "topic") -> Dict[str, Any]:
    """Transform a single item to include id and type."""
    import uuid
    
    result = {
        "id": str(uuid.uuid4()),
        "title": item.get("title", "Untitled"),
        "type": item.get("type", default_type),  # <- Может быть "topic" из JSON
    }
    
    if "description" in item:
        result["description"] = item["description"]
    
    if "children" in item and isinstance(item["children"], list):
        result["children"] = [
            self._transform_item(child, "theory")  # <- ВСЕГДА type="theory"!
            for child in item["children"]
        ]
    
    return result
```

**Проблема:**
```python
self._transform_item(child, "theory")  # <- ВСЕГДА type="theory"!
```

**Даже если AI создаст вложенные topics**, они будут принудительно преобразованы в `type="theory"` при обработке children!

**Исправление:**
```python
def _transform_item(self, item: Dict[str, Any], parent_had_children: bool = False) -> Dict[str, Any]:
    """
    Transform a single item to include id and type.
    
    Logic:
    - If item has explicit type - use it
    - If item has children - it's a topic (sub-topic)
    - If parent had children and item doesn't have explicit type - it's a topic
    - Otherwise - use default_type (theory)
    """
    import uuid
    
    # Determine type based on explicit type, children presence, or parent context
    explicit_type = item.get("type", "")
    has_children = "children" in item and isinstance(item["children"], list) and len(item["children"]) > 0
    
    if explicit_type:
        node_type = explicit_type
    elif has_children:
        # Has children = is a topic/subtopic
        node_type = "topic"
    elif parent_had_children:
        # Parent had children (was a topic), so this is a sub-topic
        node_type = "topic"
    else:
        # No children and parent didn't have children = it's a lesson
        node_type = item.get("type", "theory")
    
    result = {
        "id": str(uuid.uuid4()),
        "title": item.get("title", "Untitled"),
        "type": node_type,
    }
    
    if "description" in item:
        result["description"] = item["description"]
    
    if has_children:
        result["children"] = [
            self._transform_item(child, True)  # Pass True: parent has children
            for child in item["children"]
        ]
    
    return result
```

## Проблема #2: Пустой practice контент

### Описание
При генерации практического контента для урока возвращается пустой результат.

**Лог:**
```
WARNING - Empty practice content for node 9e40118d-b3e5-4ced-a3b9-5414b945ee9f
```

**Симптомы:**
- Theory генерируется успешно
- Practice возвращает пустой task/solution
- AI Framework получает пустой `theory_content`

### Корневые причины

#### Причина 1: LessonContentRequest не передаёт theory_content

**Файл:** `backend/app/api/ai_proxy.py`

**Текущий код:**
```python

```

**Проблема:** Frontend отправляет запрос на практику БЕЗ `theory_content`, и оно остаётся пустым.

**Исправление:** Добавить логику в `generate_lesson_content` для автоматического получения theory_content из sibling-узла:

```python
async def _find_sibling_theory(
    db: AsyncSession,
    node_id: str,
    context: dict
) -> Optional[Node]:
    """Find the theory sibling of a practice node in the same parent topic."""
    try:
        node_uuid = uuid.UUID(node_id)
    except ValueError:
        return None
    
    # Get current node
    result = await db.execute(select(Node).where(Node.id == node_uuid))
    node = result.scalar_one_or_none()
    if not node:
        return None
    
    # If node is theory - return it
    if node.type == "theory":
        return node
    
    # If node is practice - find sibling theory with same parent
    if node.type == "practice" and node.parent_id:
        siblings_result = await db.execute(
            select(Node)
            .where(
                Node.parent_id == node.parent_id,
                Node.type == "theory"
            )
            .order_by(Node.order_index)
            .limit(1)
        )
        return siblings_result.scalar_one_or_none()
    
    return None
```

## Проблема #3: Неправильная навигация theory/practice

### Описание
Кнопки "Go to Practice" и "Back to Theory" на страницах уроков ссылаются на ту же страницу, а не на связанный урок.

**Текущее поведение:**
```jsx
// theory/page.tsx
<Link href={`/lesson/${lessonId}/practice`}>
    Go to Practice
</Link>

// practice/page.tsx
<Link href={`/lesson/${lessonId}/theory`}>
    Back to Theory
</Link>
```

**Проблема:** 
- `lessonId` - это ID текущего урока
- Если мы на Theory уроке и хотим перейти к Practice - нужно перейти к sibling Practice уроку
- Если мы на Practice уроке и хотим вернуться к Theory - нужно перейти к sibling Theory уроку

**Дополнительная проблема:** 
В одной теме может быть:
- 10 теорий и 1 практика
- 5 теорий и 5 практик
- 1 теория и 10 практик (усложнённые задания)

Нет жёсткой пары 1:1 между theory и practice.

### Решение: Topic-Based Navigation

#### Вариант A: Убрать кнопки, полагаться на Prev/Next
Убрать кнопки "Go to Practice" / "Back to Theory", пользователь использует Prev/Next навигацию.

**Плюсы:** Простота, не нужно менять backend
**Минусы:** Менее очевидно для новых пользователей

#### Вариант B: Collapsible Theory Preview в Practice
Practice страница показывает краткое содержание Theory в collapsible секции.

**Плюсы:** Контекст всегда доступен
**Минусы:** Смешивание логики

#### Вариант C (рекомендуемый): Topic-Based навигация

**Frontend Theory Page (`theory/page.tsx`):**

Убрать кнопку "Go to Practice", заменить на навигацию по текущей теме:

```tsx
// В начале компонента добавить состояние для siblings
const [topicLessons, setTopicLessons] = useState<CourseNode[]>([])

// В useEffect добавить:
useEffect(() => {
    // Получить все уроки текущей темы
    const siblings = course?.nodes.filter(n => {
        if (n.type === "topic") return false
        // Проверяем тот же parent_id
        return n.parent_id === node.parent_id
    }) || []
    
    // Сортируем по order_index
    siblings.sort((a, b) => a.order_index - b.order_index)
    setTopicLessons(siblings)
}, [course, node])

// Убрать кнопку "Go to Practice" из header
// Вместо неё показывать уроки темы:
{topicLessons.length > 1 && (
    <div className="mt-6 p-4 bg-slate-800/50 rounded-lg">
        <h3 className="text-sm font-medium text-slate-400 mb-2">
            Другие уроки темы "{{parent_topic_title}}"
        </h3>
        <div className="flex flex-wrap gap-2">
            {topicLessons.map(sibling => (
                <Link
                    key={sibling.id}
                    href={`/lesson/${sibling.id}/${sibling.type}`}
                    className={cn(
                        "px-3 py-1.5 rounded-full text-sm",
                        sibling.id === lessonId
                            ? "bg-violet-600 text-white"
                            : "bg-slate-700 text-slate-300 hover:bg-slate-600"
                    )}
                >
                    {sibling.type === "theory" ? "📖" : "💻"} {sibling.title}
                </Link>
            ))}
        </div>
    </div>
)}
```

**Frontend Practice Page (`practice/page.tsx`):**

```tsx
// Добавить состояние для теории
const [relatedTheory, setRelatedTheory] = useState<{id: string; title: string; content: string} | null>(null)
```
