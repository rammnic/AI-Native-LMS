# AI-Native LMS — Тест-чеклист для Dev-режима

> Дата: 2026-03-20
> Версия: 0.4.1

---

## Как запустить

```bash
# Запустить все сервисы
docker-compose up -d

# Проверить статус
docker-compose ps
```

**Ожидаемые порты:**
- Frontend: http://localhost:3000
- Backend: http://localhost:8001
- AI Framework: http://localhost:8000
- PostgreSQL: localhost:5432

## Тест 1: Создание курса

### Ожидание
При создании курса через модалку "Create New Course" курс должен создаваться **с нодами** (структурой), а не быть пустым.

### Шаги
1. Открыть http://localhost:3000/login
2. Войти (или зарегистрироваться)
3. Нажать "Create New Course"
4. Ввести промпт: "I want to learn Python basics"
5. Выбрать difficulty: "intermediate"
6. Нажать "Generate Course"

### Результат
- [x] Появляется loading экран "Generating Course Structure"
- [x] После генерации показывается экран "Course Created!"
- [x] Кнопка "Go to Course" перенаправляет на страницу курса
- [ ] На странице курса отображается **дерево нод** (темы, теория, практика) - NOPE, пусто, в "готовом" курсе после нажатия "Generate Course Structure" что-то происходит но страница все также пустая, "Generate More" ведет себя аналогично
- [ ] НЕ показывается сообщение "No content yet" - показывается

## Тест 2: Кнопка "Generate Course Structure"

### Ожидание
На пустой странице курса (если нод нет) кнопка "Generate Course Structure" должна работать и генерировать структуру.

### Шаги
1. Создать пустой курс (или удалить все ноды в БД)
2. Открыть страницу курса
3. Нажать кнопку "Generate Course Structure"

### Результат
- [x] Кнопка показывает loading state (спиннер + "Generating...")
- [ ] После генерации появляется дерево нод
- [ ] Ноды отображаются корректно (темы, уроки)

## Тест 3: Кнопка "Generate More"

### Ожидание
Кнопка "Generate More" в хедере страницы курса должна генерировать дополнительные ноды.

### Шаги
1. Открыть страницу курса с существующими нодами
2. Нажать кнопку "Generate More" в хедере

### Результат
- [x] Кнопка показывает loading state
- [ ] После генерации количество нод увеличивается
- [ ] Новые ноды корректно отображаются в дереве

## Тест 4: Навигация по курсу

### Ожидание
Клик на ноду типа "theory" или "practice" должен перенаправлять на соответствующую страницу урока.

### Шаги
1. Открыть страницу курса с нодами
2. Раскрыть дерево (клик на topic)
3. Кликнуть на кнопку "Theory" или "Practice" рядом с нодой

### Результат
- [ ] Переход на страницу /lesson/{node_id}/theory
- [ ] Переход на страницу /lesson/{node_id}/practice

## Известные ограничения

- AI работает в mock режиме (всегда возвращает "Python для начинающих")
- Контент уроков (теория/практика) генерируется (в real AI режиме через ai_proxy ↔ pipelines контракт)
- Нет реального LLM API (нужен OPENROUTER_API_KEY)

---

*Для полного тестирования с реальным AI нужно добавить OPENROUTER_API_KEY в .env и перезапустить контейнеры.*

## Доп. Тест (после real-AI исправления): Контент Theory/Practice

### Шаги
1. Создать курс
2. Открыть урок theory → дождаться генерации → убедиться, что markdown отображается
3. Перезагрузить страницу → контент должен остаться
4. Открыть practice → увидеть task/solution/tests

### Ожидание
- Теория сохраняется в node.content (content_status = generated)
- Практика сохраняется в node.data (task/solution/tests)
